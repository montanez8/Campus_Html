Common Language Specifications

Las Common Language Specifications (CLS) son una especificación relacionada con el Common Language Runtime (CLR) en el framework .NET. La CLS define un conjunto de reglas y estándares que todas las implementaciones de lenguajes .NET deben seguir para garantizar la interoperabilidad entre ellos. Estas reglas incluyen la definición de tipos comunes, convenciones de nomenclatura, manejo de excepciones y otros aspectos del diseño de código. Al seguir las CLS, los lenguajes .NET pueden interactuar entre sí de manera más fluida y permitir la reutilización de código entre diferentes lenguajes.

La Especificación del lenguaje común es un conjunto de características básicas del lenguaje (construcciones y restricciones) que sirve como guía para los escritores de bibliotecas y compiladores. Permite que las bibliotecas sean totalmente utilizables desde cualquier idioma que admita CLS y que esos idiomas se integren entre sí. La especificación del lenguaje común es un subconjunto del sistema de tipos común. El CLS es en realidad un conjunto de restricciones del CTS. El CLS define no sólo los tipos permitidos en llamadas externas, sino las reglas para su uso, dependiendo del objetivo del usuario.

Asp .Net

ASP.NET es un framework desarrollado y comercializado por Microsoft que se utiliza para construir aplicaciones web dinámicas, aplicaciones web y servicios web XML. Apareció en enero de 2002 como la tecnología sucesora de las páginas de servidor Active Server Pages (ASP). ASP.NET está construido sobre el Common Language Runtime (CLR) de .NET Framework, lo que permite a los programadores utilizar varios lenguajes de programación, como C# y Visual Basic, para desarrollar aplicaciones web. Proporciona una amplia gama de características y funcionalidades para facilitar el desarrollo y la implementación de aplicaciones web escalables y seguras.

Web forms

Web Forms es una tecnología de Microsoft que te permite crear aplicaciones web dinámicas y funcionales usando HTML, código de cliente, código de servidor y controles de servidor. Web Forms forma parte del marco de trabajo de ASP.NET, que te ofrece muchas ventajas como un entorno administrado, la seguridad de tipos, la herencia y el soporte para varios lenguajes como Visual Basic o C#.

Con Web Forms, puedes diseñar la interfaz de usuario de tu aplicación web arrastrando y soltando controles de servidor en Visual Studio, el entorno de desarrollo integrado de Microsoft. Estos controles te permiten acceder a datos, mostrar información, validar entradas, navegar entre páginas y mucho más. También puedes escribir código de servidor para controlar la lógica de la página y responder a los eventos generados por los controles o por el usuario.

Web Forms es uno de los cuatro modelos de programación que puedes usar para crear aplicaciones web ASP.NET, junto con ASP.NET MVC, ASP.NET Web Pages y ASP.NET Single Page Applications. Cada uno de estos modelos tiene sus propias características y ventajas, y puedes elegir el que mejor se adapte a tus necesidades y preferencias.

Web Services

Los servicios web son componentes en un servidor web que una aplicación cliente puede invocar mediante solicitudes HTTP a través de la web. Los servicios web permiten comunicar e intercambiar datos o información entre diferentes aplicaciones creadas en el mismo o diferente lenguaje de programación o plataforma. Los servicios web usan una combinación de protocolos y estándares abiertos, como XML, SOAP, WSDL y UDDI, para facilitar la interoperabilidad entre las aplicaciones.

.NET es un marco de trabajo de Microsoft que te permite crear y consumir servicios web usando diferentes lenguajes como C#, Visual Basic o F#. .NET ofrece varias tecnologías para trabajar con servicios web, como ASP.NET Web Services, WCF, ASMX y RESTful. Cada una de estas tecnologías tiene sus propias características y ventajas, y puedes elegir la que mejor se adapte a tus necesidades y preferencias.

Windows Forms

Windows Forms es una tecnología de interfaz de usuario para .NET, un conjunto de bibliotecas administradas que simplifican las tareas comunes de las aplicaciones, como leer y escribir en el sistema de archivos. Con Windows Forms, puedes crear aplicaciones de escritorio para Windows con una interfaz gráfica de usuario (GUI) que incluye botones, menús, cuadros de texto, cuadros de diálogo y otros elementos. Windows Forms también te permite usar controles personalizados o de terceros para mejorar la apariencia y la funcionalidad de tus aplicaciones.

Hay dos implementaciones de Windows Forms:

1. La implementación de código abierto hospedada en GitHub. Esta versión se ejecuta en .NET 6 y versiones posteriores. El diseñador visual de Windows Forms requiere, como mínimo, Visual Studio 2019 versión 16.81. La versión más reciente es Windows Forms para .NET 7 con Visual Studio 2022, versión 17.4

2. La implementación de .NET Framework 4 compatible con Visual Studio 2022, Visual Studio 2019 y Visual Studio 2017. .NET Framework 4 es una versión de .NET solo para Windows y se considera un componente del sistema operativo Windows. Esta versión de Windows Forms se distribuye con .NET Framework

ADO .Net

es un conjunto de clases que exponen servicios de acceso a datos para programadores de .NET Framework. ADO.NET ofrece abundancia de componentes para la creación de aplicaciones de uso compartido de datos distribuidas. Constituye una parte integral de .NET Framework y proporciona acceso a datos relacionales, XML y de aplicaciones.

ADO.NET separa el acceso a datos de la manipulación de datos en componentes discretos que se pueden usar por separado o en conjunto. ADO.NET incluye proveedores de datos .NET Framework para conectarse a una base de datos, ejecutar comandos y recuperar resultados. Esos resultados se procesan directamente, se colocan en un objeto DataSet de ADO.NET para exponerlos al usuario de forma ad hoc, se combinan con datos de múltiples fuentes o se pasan entre capas. El objeto DataSet también se puede usar independientemente de un proveedor de datos .NET Framework para administrar datos locales a la aplicación o provenientes de XML2

ADO.NET proporciona funcionalidad similar a los desarrolladores que escriben código administrado que la proporcionada a los desarrolladores nativos de componentes COM por ActiveX Data Objects (ADO). Recomendamos que use ADO.NET, no ADO, para acceder a datos en sus aplicaciones .NET. ADO.NET proporciona el método más directo de acceso a datos dentro del .NET Framework. Para una abstracción de nivel superior que permita a las aplicaciones trabajar con un modelo conceptual en lugar del modelo de almacenamiento subyacente, consulte Entity Framework
